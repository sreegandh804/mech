ivy_mech.mat_pose_to_quaternion_pose(matrix, f=None)[source]¶

Convert matrix pose \(\mathbf{P}\in\mathbb{R}^{3×4}\) to quaternion pose \(\mathbf{p}_{q} = [\mathbf{x}_c, \mathbf{q}] = [x, y, z, q_i, q_j, q_k, q_r]\).

Parameters

  • matrix (array) – Matrix pose [batch_shape,3,4]

  • f (ml_framework, optional) – Machine learning framework. Inferred from inputs if None.

Returns

Quaternion pose [batch_shape,7]

ivy_mech.euler_pose_to_quaternion_pose(euler_pose, convention='zyx', batch_shape=None, f=None)[source]¶

Convert :math: Euler angle pose \(\mathbf{p}_{abc} = [\mathbf{x}_c, \mathbf{θ}_{xyz}] = [x, y, z, ϕ_a, ϕ_b, ϕ_c]\) to quaternion pose \(\mathbf{p}_{q} = [\mathbf{x}_c, \mathbf{q}] = [x, y, z, q_i, q_j, q_k, q_r]\).

[reference]

Parameters

  • euler_pose (array) – Euler angle pose [batch_shape,6]

  • convention (str, optional) – The axes for euler rotation, in order of L.H.S. matrix multiplication.

  • batch_shape (sequence of ints, optional) – Shape of batch. Inferred from inputs if None.

  • f (ml_framework, optional) – Machine learning framework. Inferred from inputs if None.

Returns

Quaternion pose [batch_shape,7]

ivy_mech.increment_quaternion_pose_with_velocity(quat_pose, quat_vel, delta_t, f=None)[source]¶

Increment a quaternion pose \(\mathbf{p}_{q} = [\mathbf{x}_c, \mathbf{q}] = [x, y, z, q_i, q_j, q_k, q_r]\) forward by one time-increment \(Δt\), given the velocity represented in quaternion form \(\mathbf{V}_q\). Where \(\mathbf{P}_q (t+1) = \mathbf{P}_q(t) × \mathbf{V}_q\), \(t\) is in seconds, and \(×\) denotes the hamilton product.

[reference]

Parameters

  • quat_pose (array) – Quaternion pose. [batch_shape,7]

  • quat_vel (array) – Quaternion velocity. [batch_shape,7]

  • delta_t (float) – Time step in seconds, for incrementing the quaternion pose by the velocity quaternion.

  • f (ml_framework, optional) – Machine learning framework. Inferred from inputs if None.

Returns

Incremented quaternion pose [batch_shape,7]
