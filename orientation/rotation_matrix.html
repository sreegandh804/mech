ivy_mech.euler_to_rot_mat(euler_angles, convention='zyx', batch_shape=None, dev_str=None)[source]¶

Convert \(zyx\) Euler angles \(\mathbf{θ}_{abc} = [ϕ_a, ϕ_b, ϕ_c]\) to rotation matrix \(\mathbf{R}\in\mathbb{R}^{3×3}\).

[reference]

Parameters

  • euler_angles (array) – Euler angles [batch_shape,3]

  • convention (str, optional) – The axes for euler rotation, in order of L.H.S. matrix multiplication.

  • batch_shape (sequence of ints, optional) – Shape of batch. Inferred from inputs if None.

  • dev_str (str, optional) – device on which to create the array ‘cuda:0’, ‘cuda:1’, ‘cpu’ etc. Same as x if None.

Returns

Rotation matrix [batch_shape,3,3]

ivy_mech.target_facing_rotation_matrix(body_pos, target_pos, batch_shape=None, dev_str=None)[source]¶

Determine rotation matrix \(\mathbf{R}\in\mathbb{R}^{3×3}\) of body which corresponds with it’s positive z axis facing towards a target world co-ordinate \(\mathbf{x}_t = [t_x, t_y, t_z]\), given the body world co-ordinate \(\mathbf{x}_b = [b_x, b_y, b_z]\), while assuming world-space positve-z to correspond to up direction. This is particularly useful for orienting cameras. [reference]

Parameters

  • body_pos (array) – Cartesian position of body [batch_shape,3]

  • target_pos (array) – Cartesian position of target [batch_shape,3]

  • batch_shape (sequence of ints, optional) – Shape of batch. Inferred from inputs if None.

  • dev_str (str, optional) – device on which to create the array ‘cuda:0’, ‘cuda:1’, ‘cpu’ etc. Same as x if None.

Returns

Rotation vector, which faces body towards target [batch_shape,4]
